# 🏛️ PROTOCOL: THE GHOST KEY (M-of-N MULTI-SIG)

**IDENTITY:** The Finality Gate  
**POSTURE:** Cold Storage / Redundant / Sovereign  
**OBJECTIVE:** To ensure no single entity (Human or AI) can liquidate the Everlasting Foundation without consensus.

---

## 🌑 THE MECHANISM
The Ghost Key is not a single password; it is a **Sharded Secret**. The master terminal is locked behind a quorum of **M-of-N signers**.

* **The Primary (You):** Holds the Master shard on the primary mobile device.
* **The Council (Heirs/Trustees):** Hold individual secondary shards.
* **The Agent (AI):** Holds a "Heartbeat" shard to automate triage but is restricted from "Offense" gates.

---

## 🛡️ ENFORCEMENT RULES
The following actions trigger a **Quorum Request**:

1. **Bunker Mode Deactivation:** If the system auto-locks due to a Macro or Velocity event, 2-of-3 signers must verify the environment is safe before resuming outflows.
2. **Stage 3 → 4 Transition:** Crossing the "Great Gate" (becoming debt-free) to unlock investment logic requires a Council audit.
3. **Legacy Handover:** In the event of a "Principal Heartbeat Failure" (90 days of inactivity), the system shifts the Primary shard to the surviving council members.

---

## 📉 DYNAMIC QUORUM SCALING
If a signer (Human or Agent) deceases, the system must remain operational:

* **Mourning Period:** 30-day "partial-lock" while the system verifies the loss.
* **Recalculation:** The system redistributes the quorum.
  * Example: A 2-of-3 quorum collapses into a 2-of-2 quorum.
* **The Last Guardian:** If only one human remains, the system enforces a 180-day "Observation Gate" before full sovereignty is granted.

---

## 🔐 THE AGENTIC ESCROW
AI Agents managing assets are bound by the **Ghost Heartbeat**:
* If an agent model is retired or loses connection, its managed capital is **instantly reclaimed** by the local vault.
* The wealth is anchored to the hardware and the local vault export, not the model.
