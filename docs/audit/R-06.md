# R-06 — Undocumented Interfaces

**Category:** Documentation / Architecture
**Severity:** Normal

## Description

All exported functions across the critical modules — agent API, database, security, and alerts — had no JSDoc. Parameters, return types, error conditions, and behavioral contracts were undocumented. Consumers of these interfaces had no generated documentation and no specification of expected inputs and outputs beyond reading the source.

## Impact

API consumers could not determine function contracts without reading source code. IDEs could not provide autocomplete or type hints. Generated documentation (Storybook, Typedoc) was empty. Onboarding for new contributors was slowed. The absence of documentation made it impossible to verify whether the interface and its implementation agreed.

## Evidence

- `src/agents/api.ts` — no JSDoc
- `src/agents/api.types.ts` — no JSDoc
- `src/db/sovereignDB.js` — no JSDoc
- `src/security/vaultSigning.js` — no JSDoc
- `src/security/passkeys.js` — no JSDoc
- `src/crypto/sovereignCrypto.js` — no JSDoc
- `src/alerts/adapters.js` — no JSDoc

## Required Fix

Add JSDoc to all exported functions. Each JSDoc block must specify: `@param` with type and meaning, `@returns` with type and meaning, `@throws` for known error conditions, and `@description` for behavioral notes. Do not add JSDoc to internal functions.
