# R-01 — Missing Type Safety in Agent Core

**Category:** Security / Architecture
**Severity:** Critical

## Description

The agent core (`src/agents/`) operated with no type definitions. Request payloads, capability checks, and response shapes were all `any` or untyped. Invalid requests could not be rejected at the type layer.

## Impact

Unvalidated data flowed through the agent request pipeline. Malformed or adversarial payloads could bypass behavioral checks because there was no type-level enforcement. Runtime type errors were caught late, in业务 logic, rather than at the boundary.

## Evidence

- `src/agents/api.ts` — all functions took/returned untyped objects
- `src/agents/api.types.ts` — did not exist
- No `ValidationErrorCodes` enum
- No `validateAgentRequest()` function

## Required Fix

Introduce strict TypeScript types for all agent request/response boundaries. Define `CAPABILITIES`, `ValidationErrorCodes`, `APIResult<T>`, and `AgentRequest` interfaces. Implement `validateAgentRequest()` as a typed guard at the entry point.
