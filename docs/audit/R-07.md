# R-07 — Execution Environment Non-Determinism

**Category:** Reliability / Build Integrity
**Severity:** High

## Description

The system previously allowed multiple package managers, inconsistent lockfiles, and environment drift between local and CI execution. Dual lockfiles existed (`package-lock.json` and `pnpm-lock.yaml`), creating ambiguous authority. The CSP `unsafe-eval` directive was present in production, valid only in development with Vite HMR.

## Impact

Non-reproducible builds. Inconsistent verifier outcomes across environments. Inability to trust execution results when run locally vs in CI. Environment state could diverge silently between machines.

## Evidence

- Dual lockfiles: `package-lock.json` + `pnpm-lock.yaml` coexisted
- `unsafe-eval` present in `index.html` CSP (Vite dev-only, not production-safe)
- Inconsistent install paths between npm and pnpm
- No enforcement of which package manager was canonical

## Required Fix

Single package manager enforcement with declared lockfile authority. Production CSP uses `script-src 'self'` only. CI environment must be aligned with local. Deterministic install and build process with reproducible outputs.

## Mapped Tasks

- ENVIRONMENT_FIX-01 — Duplicate App cleanup (removed dual-lockfile ambiguity, kept `pnpm-lock.yaml`)
- ENVIRONMENT_FIX-02 — CSP + environment stabilization (`unsafe-eval` removed, production CSP hardened)

## Status

**RESOLVED** — `package-lock.json` removed, `pnpm-lock.yaml` authoritative. `unsafe-eval` removed from production CSP. Committed `1e85b362`. Build verified without `unsafe-eval`. pnpm used consistently.
